# Post-Phase-1 Desktop R2 Task 3 Scope

## Goal

This task hardens Desktop launch ergonomics. It does not expand Desktop into a heavier desktop product.

The target is a lower-friction daily entry:

- a shorter repo-root launch command
- clearer separation between recommended and lower-level entry paths
- more readable startup feedback
- a more direct “open shell and enter the shared workbench” experience

## In Scope

- Desktop launch entry ergonomics
- Desktop startup feedback wording
- Desktop launch-related tests
- Desktop launch-related documentation

## Out Of Scope

This task does not:

- add native business pages
- add Desktop-owned business logic
- add installers or auto-updaters
- add packaging or distribution platforms
- add WebView-based workbench embedding
- change the shared workbench role

## Boundary Reminder

Desktop remains a shell-level entry:

- it opens the shared Web workbench
- it remains outside the business logic center
- this task improves launch ergonomics only
