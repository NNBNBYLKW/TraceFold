# Step 9 Chapter 3 Scope

## Purpose

Step 9 Chapter 3 aligns terminology, state wording, layer presentation, and interaction language across Web, Desktop, Telegram, and user-facing API semantics.

This chapter is not UI polish. It is a semantics and interaction consistency pass so the current system reads and behaves like one system.

## In Scope

- Workbench home wording and role language
- Dashboard summary wording
- Pending list/detail/action wording
- Expense / Knowledge / Health list/detail wording
- Rule alert and AI derivation layer wording
- Desktop minimal service/workbench labels
- Telegram formatter and command feedback wording
- User-facing state phrases such as loading, empty, unavailable, not generated, disabled

## Out of Scope

- Large visual redesign
- New design system
- New business capability
- Workbench capability expansion
- Desktop runtime expansion
- Telegram workflow expansion
- Business-logic changes for the sake of copy consistency

## Success Criteria

- Same concept uses the same core language across entry surfaces
- Same state uses the same core meaning across pages and entry surfaces
- Formal fact / rule alert / AI derivation remain visually and verbally distinct
- Workbench home keeps Template / Shortcut / Recent / Dashboard roles clear
- Web, Desktop, and Telegram feel like different surfaces of one system, not different products
