# Step 9 Chapter 5 Drift Trigger Table

| drift_direction | early_trigger | stronger_trigger | treat_as_drift_when |
| --- | --- | --- | --- |
| 万能平台化 | 文档开始把 Workbench 说成“容纳任何工作”的平台 | 新模块被按“都能放进去”逻辑吸入首页 | control layer 被默认当成 universal workspace |
| 自动化引擎化 | Template payload 开始谈 actions/tasks | Shortcut 变执行按钮，出现 workflow/script wording | Template/Shortcut 不再只是入口上下文 |
| AI 放权扩大化 | AI 文案越来越像系统结论 | AI 自动确认、自动修正、自动写正式事实 | AI 不再只是 derivation/explanation |
| 图谱中心化 | 文档把 graph 视图写成产品中心 | IA 或导航以 graph 为主轴重排 | Knowledge 不再服务统一工作台，而是反过来主导它 |
| Health 医疗工具化 | 规则或 AI 文案像医疗判断 | 页面或通知暗示诊断/治疗建议结论 | Health 不再只是记录、提醒、回顾 |
| Desktop 入口中心漂移 | shell 文档开始像桌面客户端 spec | 出现桌面独立业务页、桌面写路径、桌面模板管理 | Desktop 不再只是 shell |
| Telegram 管理端漂移 | Bot 文档开始出现 mode/template 管理措辞 | 新增模板 CRUD、workbench 命令面、复杂 review state machine | Telegram 不再只是 lightweight adapter |
| 外部工具附庸化 | 出现“先兼容某工具再说”的默认口径 | 目录结构、数据模型、IA 为外部工具形态让路 | TraceFold 主轴被外部工具重塑 |
| 工程规模膨胀 | 注释/脚本开始默认 queue/cache/plugin/microservice | 这些能力被写进当前阶段默认计划 | Phase 1 被默许跨到平台工程化扩张 |

## Freeze Rule

如果上表中的 stronger trigger 出现，而文档没有明确重新开阶段边界，应视为 Phase 1 偏航。
